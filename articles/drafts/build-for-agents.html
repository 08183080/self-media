【AI智能体】卡帕西一句话点醒所有人，不为Agent建造的产品将被淘汰

Andrej Karpathy 最近发了一段话，只有短短几行，却像一把手术刀，精准切开了整个行业最容易被忽视的一层真相。

他说：CLI 之所以令人兴奋，恰恰是因为它是一种"遗留"技术 —— 这意味着 AI 智能体可以原生地使用它们、组合它们、通过整个终端工具链与它们交互。

这句话的深意，远超表面。

旧世界的接口，新世界的入口

我们花了二十年时间，把一切都做成了图形界面。按钮越做越圆，动画越做越流畅，交互越做越"人性化"。但这些精心设计的界面有一个致命的盲区：它们是为人类的眼睛和手指设计的，不是为智能体的代码和推理设计的。

一个精美的 Web 控制台，对 AI 来说是一面墙。一个朴素的命令行接口，对 AI 来说是一扇门。

这就是卡帕西看到的反直觉真相：技术栈中越"古老"的层，反而越容易被 AI 接管。 命令行、纯文本、管道操作、标准输入输出 —— 这些 Unix 哲学的基石，在 AI 智能体的视角下，变成了最优雅的 API。

三分钟的颠覆

卡帕西举了一个例子：让 Claude 安装 Polymarket 的 CLI，然后要求它构建一个展示最高交易量预测市场及 24 小时变化的终端仪表盘。

整个过程只用了大约三分钟。

不是三天，不是三小时，是三分钟。而且你还可以让它变成 Web 应用、桌面应用、或者任何你想要的形态。更强大的是，你可以把它当作更大流水线中的一个模块。

这不是炫技。这是一种新的构建范式正在成型：人类提出意图，智能体组装工具，工具通过标准接口协作。 整个过程中，那些"过时"的 CLI 工具成了智能体手中最趁手的积木。

同样地，GitHub CLI 也是如此 —— 智能体可以用它浏览仓库、查看 Issue、Pull Request、Discussion，甚至直接阅读代码本身。不需要打开浏览器，不需要登录网页，不需要任何图形界面。纯粹的数据流动，纯粹的程序化交互。

产品的灵魂拷问

卡帕西抛出了一个每个产品负责人都该认真思考的问题：

如果你有任何产品或服务，想一想：智能体能访问和使用它们吗？

然后他列出了一份检查清单：

  • • 你的文档能导出为 Markdown 吗？ —— 不是 PDF，不是富文本，是 Markdown。因为 Markdown 是 AI 的母语。
  • • 你为产品编写了 Skills 吗？ —— Skills 是智能体理解和操作你产品的说明书，告诉 AI "我能做什么、怎么调用"。
  • • 你的产品可以通过 CLI 或 MCP 使用吗？ —— 如果答案是否，那你的产品对智能体来说基本不存在。

这份清单看似简单，背后是一场深刻的范式转移：你的用户不再只是人类了。

Markdown 是 AI 的母语

为什么卡帕西特别提到 Markdown？因为在所有的文档格式中，Markdown 是最接近"纯信息"的存在。它没有复杂的样式层，没有二进制编码，没有需要专门解析器的私有格式。

一份 Markdown 文档可以被任何文本工具处理，可以被版本控制系统追踪，可以被 AI 直接阅读和生成。它是人类和机器之间的最小公约数。

而那些锁在 Notion 数据库里、藏在 Confluence 富文本编辑器中、嵌在 PDF 分页符之间的知识？对智能体来说，那是被关在保险箱里的信息，钥匙丢了。

MCP：智能体的万能插座

卡帕西提到的 MCP（Model Context Protocol），是另一个关键的基础设施。如果说 CLI 是智能体的双手，MCP 就是智能体的神经系统。

MCP 提供了一个标准化的方式，让 AI 模型能够连接到外部工具和数据源。它不像传统的 API 集成那样需要针对每个服务写定制代码，而是提供了一个统一的协议层，让智能体能够"即插即用"地接入各种能力。

想象一下：一个智能体通过 MCP 同时连接了你的代码仓库、项目管理工具、监控系统和文档库。它可以在一次对话中，从 Issue 出发，阅读相关代码，提交修复，更新文档，关闭工单。整个流程不需要你切换任何一个界面。

真正的竞争壁垒正在重新定义

如果我们把卡帕西的观点推到极致，会得到一个令人不安的结论：

在智能体时代，产品的竞争力不再仅仅取决于它对人类有多好用，还取决于它对 AI 有多好用。

一个功能强大但只有图形界面的 SaaS 产品，在智能体面前可能不如一个功能简单但有完善 CLI 和 MCP 支持的开源工具。因为后者可以被编排、被组合、被嵌入到自动化流水线中，而前者只能等着人类一个按钮一个按钮地去点。

这不是说图形界面会消失。而是说，如果你的产品只有图形界面，你正在把一半的潜在"用户"拒之门外 —— 而且这一半的增长速度远超另一半。

Build. For. Agents.

卡帕西用三个词结束了他的分享：

It's 2026. Build. For. Agents.

这不是一句口号，而是一个关于生存的判断。

CLI、Markdown、Skills、MCP —— 这些词汇听起来都不性感。它们没有"大模型"那样的光环，没有"AGI"那样的想象空间，没有"多模态"那样的视觉冲击力。

但它们是管道、是接线板、是螺丝钉。没有它们，再强大的 AI 模型也只是一个困在对话框里的天才 —— 什么都懂，什么都做不了。

2026 年的真相是：AI 的能力已经不是瓶颈了，AI 能触达的范围才是瓶颈。 而扩大这个范围的关键，不在于训练更大的模型，而在于为智能体铺好路、开好门、接好线。

所以，最后回到那个灵魂拷问：

你的产品，为智能体准备好了吗？
