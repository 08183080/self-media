# 【AI编程】写代码已被解决，工程师正在变成另一种生物

> 看完 Anthropic 的 Boris Cherny 在油管上的分享，有一个判断在我脑子里挥之不去：我们正在打的这场游戏，版本号已经换了。以前你叫工程师，本质是代码编写者。现在你叫 Builder，代码是 AI 写的，你负责决定建什么、不建什么。

---

## 一、从「写」到「建」：版本迭代已经发生

Boris 给出了一个非常激进的结论：**Coding 已经基本被解决了。**

他不是在说「AI 能帮点小忙」。他是说，他自己从 2025 年 11 月之后就没再手写过一行代码，但每天能提交 10 到 30 个 PR。个人生产力提升了 200%。GitHub 上约 4% 的 Commit 已经是由 Claude Code 完成的；到 2026 年底，这个数字可能会超过 20%。

这不是边际改进，这是**角色重写**。

关键在于：对话 AI 和 Agent 的本质差异。对话 AI 被关在聊天框里，你问它答，它不能动你的文件、不能跑你的终端、不能碰你的 Git。而 Agent 拥有完整的工具和计算机使用权限——能直接读写文件、运行命令、处理版本记录。**当「构建」这件事不再需要你亲手敲键盘，瓶颈就转移了。** 真正的竞争在于：决定该构建什么。

所以工程师的职能，从「把需求变成代码」变成了「把判断变成方向，让 Agent 把方向变成代码」。草木竹石，片叶飞花，皆可 AI+。写代码这件事，已经可以+出去了。

---

## 二、四个认知转变：押注通用、限制人手、边界消失、无限 Token

Boris 分享里还有几个认知，值得单独拎出来。

**第一，通用的强大模型总会战胜靠人类经验调优的特定模型。** 与其费力做微调和复杂的流程控制，不如直接押注更强的通用模型。这是第一性原理上的选择：能力的天花板在基础模型，不在你缝的那层壳。

**第二，在团队管理上，故意限制人手反而会强迫工程师极端依赖 AI，从而催生出更高效的自动化流。** 这在 Anthropic 内部被验证是有效的创新手段。听起来反直觉，但逻辑很清晰：人一多，就会有人去填坑、补位、手写；人一少，不自动化就干不完，反而倒逼出真正的 Agent 工作流。

**第三，PM、设计师、财务、数据科学家都会写代码。** 专业边界正在消失。未来的核心竞争力是产品理解力和决策能力，而不再是语法熟悉度。代码从「专业门槛」变成「通用工具」，谁都能用 Agent 产出可运行的东西，那比的就是谁更懂问题、谁更敢拍板。

**第四，在创新阶段，应该给工程师无限的 Token 额度。** 最强模型单次成本高，但一次就能做对；低端模型便宜，反复修正反而更贵。准确性带来的提效，会让实际成本倒挂。省小钱、花大时间的策略，在 AI 时代是错的。

---

## 三、Builder 时代：决定建什么的人，才叫在打游戏

所以回到那个隐喻：**我们正在打的游戏，规则已经换了。**

以前你的等级是「能写多少行、能扛多少需求」。现在你的等级是「能定义多少有价值的问题、能做出多少不返工的决策」。写代码已经被解决，没被解决的是：什么是值得建的、先建什么、不建什么。

Boris 零手写代码、日更数十 PR，不是因为他会魔法，而是因为他站在了「Builder」这一侧：清楚目标、拆得开任务、敢交给 Agent、敢验收结果。世间观变了，世界就变了。你把工程师还定义成「写代码的人」，你就还在旧版本里；你把自己定义成「决定建什么并让机器建出来的人」，你就已经在新版本里了。

理论上应该存在的事情，99% 的可能它已经存在了。Builder 时代已经存在了，只是分布还不均匀。请变得狂喜，我们去把它变成自己的日常吧。
