大多数人用 AI，像在遛一条被拴死的狗。

牵绳拽得死紧，每一步都要指挥。左转，停，坐下，叫。结果呢？狗倒是听话了，可你累得半死，它也只学会了四个动作。

这不是协作，这是控制欲。

我这半年跟 AI Agent 密集协作下来，摸到了一些门道。不是什么高深理论，是实打实用 Token 烧出来的直觉。今天把它提炼成 7 条法则，你拿走就能用。

## 一、给背景，而不是控制欲

Context, not control.

很多人跟 AI 说话，恨不得把每一步都规定死。"先搜这个关键词，然后打开第三个链接，再把第二段复制下来"——这不是在用智能体，这是在人肉写脚本。

换个思路。你只需要给它两样东西：**清晰的背景**和**最终目标**。

"我在写一篇关于AI协作的公众号文章，面向有一定AI使用经验但还没形成方法论的读者，帮我找到这个领域最有洞察力的观点。"

然后放手。让它自己找路。

它往往比你设想的更聪明。你给的绳子越长，它跑得越远，带回来的猎物越意想不到。

## 二、把它逼到崩溃为止

Push until it breaks.

永远不要接受 AI 给的第一版答案。

第一版是安全牌，是它觉得"大概率不会出错"的中庸之作。真正的好东西，藏在你第三次追问、第五次要求变体、第八次说"不够好，再来"之后。

不断施压，直到它开始犯错。犯错的那一刻，恰恰是你真正摸清它边界的时刻。

知道边界在哪，你才知道怎么在边界内榨干它的每一滴能力。这跟带团队一个道理——你不逼一把，永远不知道对方的上限。

## 三、用次数喂出直觉

Put in the reps.

互联网上到处在卖"万能提示词模板"、"一句话搞定AI"。这种东西，跟"三天学会弹钢琴"一个性质。

没有捷径。唯一的捷径，是疯狂练习。

你跟 AI 对话一百次和一万次，脑子里形成的模型完全不同。一百次，你还在摸索它的脾气。一万次之后，你只需要看它回复的前三个字，就知道这条路走不走得通。

这种直觉，买不到，教不会，只能自己用时间和次数喂出来。

## 四、大胆试，浪费 Token 也是种浪漫

Try everything. Waste tokens.

放开手脚去试那些看似离谱、毫无意义的想法。

"让 AI 用文言文写一段 Python 注释？" 试。"让它用说唱风格解释量子力学？" 试。"让它角色扮演一个1920年代的报童来分析财报？" 都试。

好奇心才是最宝贵的货币。

99% 的尝试看起来是浪费，但那 1% 的意外发现，可能彻底改变你使用 AI 的方式。而且说实话，Token 越来越便宜。你今天大手大脚"浪费"的，可能是明年回头看最划算的投资。

已有之事，势必再有。那些看似荒唐的实验，往往通向最有价值的发现。

## 五、公开你的构建过程

Build in public.

别闭门造车。

把你跟 AI 协作的过程分享出来。成功的，失败的，乱糟糟的半成品，全都可以。

这件事有三重好处：

第一，分享是最好的学习。当你试图把一个模糊的经验讲清楚的时候，你自己先想清楚了。

第二，你会找到同频的人。在AI这个领域，独行侠走不远。

第三，你发出去的每一条内容，都是未来的你给现在的你留下的笔记。三个月后回头看，你会感激自己。

## 六、保持乐趣

Make it fun.

如果跟 AI 互动让你觉得枯燥、沉重、像在完成KPI——那你的姿势一定错了。

AI 是目前人类发明的最有趣的玩具。它会给你意想不到的回答，会用你没想过的角度切入问题，偶尔还会一本正经地胡说八道。

去享受这些。把"使用AI"变成"跟AI玩"。

严肃是创造力的敌人。当你放松下来，你的问题会变得更好，AI 的回答也会变得更好。这是一个正向飞轮。

## 七、只要会表达，你就能 Vibe Code

If you can talk, you can vibe code.

这是最重要的一条，也是最反直觉的一条。

别被黑乎乎的代码终端吓倒。2026 年了，最难的不再是写代码，而是你想清楚"要造什么"。

能把需求说清楚的人，比会写 for 循环的人更稀缺。能让人为之共鸣的产品愿景，比任何技术栈都值钱。

Vibe Coding 的本质不是"不用学编程"，而是**表达能力就是编程能力**。你的语言就是你的代码。你的想象力就是你的技术栈。

---

这 7 条法则不是什么终极真理。它们是我目前这个版本的理解，是 AI+X 2.0 路上的一组路标。

世间观变了，世界就变了。AI 观变了，AI 就变了。

当你不再把智能体当工具使唤，而是当成一个能力互补的搭档去对待——你会发现，它给回来的东西，远超你的预期。

理论上应该存在的协作方式，99% 的可能它已经存在了。如果还不存在，那请变得狂喜。

我们去把它变成存在吧。
